CSS Preprocessors & WordPress Themes:

How Sass and Compass can make your CSS more efficient, and how they can work with WordPress theme development.

What is a preprocessor?

pre·proc·es·sor /prēˈpräsˌesər/

Noun:
A computer program that modifies data to conform with the input requirements of another program.

CSS Preprocessors

  • LESS
  • Stylus
  • Sass FTW!

Sass

  • Two formats: .sass & .scss
  • .scss is newer & easier to use
  • .scss accepts standard CSS
  • you can change normal .css files to .scss

Why bother?

Reason #1: nesting

Normal CSS


  .container{
    width: 50%
  }
  .container h1{
    margin: 1em;
  }
  .container h1 a{
    text-decoration: none;
    color: #000;
  }
  .container h1 a:hover{
    text-decoration: underline;
    color: #660000;
  }

Sass nesting

Sass (.scss syntax)


  .container{
    width: 50%
    h1{
      margin: 1em;
      a{
        text-decoration: none;
        color: #000;
        &:hover{
          text-decoration: underline;
          color: #660000;
        }
      }
    }
  }

Reason #2: variables

Normal CSS


  .container{
    background: #990000;
    color: #ddd
  }
  .box{
    background: #990000;
    color: #ddd;
  }
  .widget{
    background: #990000;
    color: #ddd;
  }

Reason #2: variables

.scss


  $red: #990000;
  $gray: #ddd;
  
  .container{
    background: $red;
    color: $gray;
  }
  .box{
    background: $red;
    color: $gray;
  }

Reason #3: mixins

Normal CSS


  .container{
    -webkit-border-radius: 5px;
    -moz-border-radius: 5px;
    -ms-border-radius: 5px;
    -o-border-radius: 5px;
    border-radius: 5px;
  }

Reason #3: mixins

.scss


  .container{
    @include border-radius(5px);
  }

Requires a mixin:


  @mixin border-radius($radius){
      -webkit-border-radius: $radius;
      -moz-border-radius: $radius;
      -ms-border-radius: $radius;
      -o-border-radius: $radius;
      border-radius: $radius;
}

Reason #4: Compass

.scss


@import "compass";

.container{
    @include border-radius(5px);
}

Compass is a huge library of common mixins.

Never worry about vendor prefixes again.

Cross broswer CSS for things like inline-block.

Automatically replace text with an image.

Automatically generate sprites and relevant CSS.

Reason #4: separate code for humans and machines

You can break up stylesheets into modules that make sense, and compile them together into one stylesheet.

CSS (.scss) becomes more readable and usable for you

the CSS that is generated can be optimized for the machine.

Single stylesheet reduces HTTP requests

Outputted CSS can be minified and compressed, while .scss files remain readable

The Catches:

  • Preprocessors require a compiler.
  • Terminal in Mac OS X
  • Codekit: http://incident57.com/codekit/

WordPress?

  • WordPress themes require a style.css file in root of theme directory
  • Requires specific information in a commented section at the top
  • Preprocessors can interfere with CSS comments
  • Use the default style.css file only for the required comment block, and include separate compiled stylesheet(s).
